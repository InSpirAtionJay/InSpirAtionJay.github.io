2. LocalMapping 成员函数说明

2.1 构造函数，通过传入的地图指针以及是否为単目地图跟踪来初始化LocalMapping,此时可接受关键帧并且认为上一次LocalMapping已经结束

2.2 设置LoopCloser以及Tracker指针

2.3 将关键帧插入到列表中进行等待

2.4 查看列表中是否有等待被插入的关键帧

2.5 处理列表中的关键帧,函数用于计算关键帧特征点的BoW映射，将关键帧插入地图。

2.6 剔除ProcessNewKeyFrame函数中引入的质量不好的MapPoints

设立观测阈值，単目为2，双目为3
遍历所有待检测的点进行剔除，剔除点的规则：

  • 1.该点已经是坏点的MapPoints直接从检查链表中删除（该点已经被删除）
  • 2.跟踪到该MapPoint的Frame数（即3D点可以找到对应的特征点）相比预计可观测到该MapPoint的Frame数（Observations可以观测到的）的比例需大于25%
    // IncreaseFound / IncreaseVisible < 25%，注意不一定是关键帧，设定badflag，并从列表中删除该点
  • 3.从该点建立开始，到现在已经过了不小于2个关键帧，但是观测到该点的关键帧数却不超过cnThObs（単目=2）帧，那么该点检验不合格，设定badflag，并从列表中删除该点

从建立该点开始，已经过了3个关键帧而没有被剔除，则认为是质量高的点，因此没有SetBadFlag()，仅从队列中删除，放弃继续对该MapPoint的检测

2.7 CreateNewMapPoints() 生成新的地图点

  • 步骤一：在当前关键帧的共视关键帧中找到共视程度最高的nn帧相邻帧vpNeighKFs

  • 步骤二：得到当前帧的位姿等相机信息，得到光心坐标，焦距等相机参数。

  • 步骤三：得到临近关键帧相机光心坐标，和当前帧的基线长度，然后和景深作比较，如果景深过深或者基线太短就放弃这一帧

  • 步骤四：根据两个关键帧的位姿计算它们之间的基本矩阵

  • 步骤五：根据这两帧计算F矩阵，根据矩阵计算匹配点索引，存在vMatchedIndices里，通过极线约束限制匹配时的搜索范围，进行特征点匹配，vector >存的是两个匹配的特征点在各自帧中的序号

  • 步骤六：得到参考帧的位姿等相机信息，得到光心坐标，焦距等相机参数。

  • 步骤七：对每对匹配通过三角化生成3D点,和Triangulate函数差不多

    • 获得矫正特征点信息

    • 利用反投影得到视差角

  • 步骤八：三角化恢复3D点

  • 步骤九：检测生成的3D点是否在相机前方，z大于0

  • 步骤十：计算重投影误差是否满足要求

  • 步骤十一：检查尺度连续性

    满足最后一个公式的尺度才满足要求

  • 步骤十一：三角化生成3D点成功，构造成MapPoint

  • 步骤十二：为该MapPoint添加属性

  • 步骤十三：将新产生的点放入检测队列

2.8 检查并融合当前关键帧与相邻帧（两级相邻）重复的MapPoints

  • 步骤一：获得当前关键帧在covisibility图中权重排名前nn的一级邻接关键帧

  • 步骤二：找到当前帧一级相邻与二级相邻关键帧，放入vpTargetKFs中，mnFuseTargetForKF标记已经加入待如何列表，避免重复加入

  • 步骤三：将当前帧的MapPoints与其他帧融合

  • 步骤四：将其他帧的MapPoints与当前帧融合

  • 步骤五：更新当前帧MapPoints的描述子，深度，观测主方向等属性

  • 步骤六：更新当前帧的MapPoints后更新与其它帧的连接关系

2.9 关键帧剔除，在Covisibility Graph中的关键帧，其90%以上的MapPoints能被其他关键帧（至少3个）观测到（需要满足尺度约束），则认为该关键帧为冗余关键帧。

  • 步骤一：根据Covisibility Graph提取当前帧的共视关键帧

  • 步骤二：提取每个共视关键帧的MapPoints

  • 步骤三：设定观测阈值，冗余地图点计数（能被另外三个关键帧观测到，同时满足尺度要求），和所有地图点计数（非坏点）

  • 步骤四：遍历该局部关键帧的MapPoints，判断是否90%以上的MapPoints能被其它关键帧（至少3个）观测到，并且该点要满足尺度约束

  • 步骤五：该局部关键帧90%以上的MapPoints能被其它关键帧（至少3个）观测到，则认为是冗余关键帧，调用PKF的SetBadFlag()函数

3. LocalMapping 主函数 Run() 函数

线程们是一起动作的，也就是说自LocalMapping被构造出来，有一些参数就已经初始化了，就已经和Tracking同时运行。
它一直在查看Tracking线程的动作，随时处理传进来的关键帧，同时Tracking也在检查由LocalMapping更新的状态考虑要不要插进新关键帧。

  • 步骤一：对于单目来说，函数中如果把标志位置为false，那么tracking线程直接就不能产生新的关键帧（Tracking中有个函数NeedNewKeyFrame 会返回false，false的话不可以生成新关键帧不可插入关键帧队列）不能传给LocalMapping线程。
    当处理完一个关键帧的时候，函数flag为true，可以接受新关键帧。

  • 步骤二：CheckNewKeyFrames()检查队列中是否还有关键帧

  • 步骤三：ProcessNewKeyFrame()从缓冲队列中取出一帧，并计算新关键帧的BOW描述子，对该关键帧的observations进行更新，更新每个地图点的共视信息，重新计算描述符，观测方向，预测尺度和深度，更新关键帧间的连接关系，Covisibility图和Essential图(tree)，并将该关键帧插入全局地图。
